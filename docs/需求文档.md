### 任务看板
基于 next.js + prisma + SQLite 全栈：看板任务管理（KanBan Board）
· 目标：实现类似 Trello 的任务拖拽（Todo/Doing/Done）及数据同步。
1. 功能模块划分
1.1 看板管理
创建新看板
编辑看板名称
删除看板
切换不同看板

1.2 列管理（待办/进行中/已完成）
添加新列
编辑列名称
删除列（级联删除列下所有任务）
拖拽调整列顺序
默认三列：Todo、Doing、Done

1.3 任务卡片管理
创建任务
任务标题（必填）
任务描述（可选）
标签选择（可选）
截止日期（可选）
编辑任务
删除任务
拖拽任务到不同列
拖拽调整同列内的任务顺序
任务详情弹窗

1.4 标签系统
创建标签
编辑标签（名称、颜色）
删除标签
为任务添加/移除标签

1.5 数据持久化
所有操作实时保存到数据库
页面刷新后数据不丢失
拖拽操作自动同步

1.6 性能优化
使用 React.memo 优化任务卡片渲染
虚拟滚动处理大量任务
API 请求防抖/节流
图片懒加载
使用 Next.js 的 App Router 实现服务端组件和客户端组件分离

2. 用户体验要求
2.1 交互反馈
拖拽时有半透明预览效果
拖拽时显示可放置区域高亮
操作成功/失败有 Toast 提示
加载状态显示骨架屏或加载动画

2.2 响应式设计
桌面端完整功能
平板端优化触摸拖拽
移动端简化视图（可考虑上下布局）

2.3 辅助功能
键盘快捷键支持
Enter：确认创建/编辑
Esc：关闭弹窗
Delete：删除任务（需确认）
屏幕阅读器支持

3. 数据验证规则
typescript
// 任务验证
- 标题：1-100个字符，不能为空
- 描述：最多1000个字符
- 截止日期：不能早于当前日期

// 列验证
- 列名：1-50个字符，不能为空
- 同一看板下列名不能重复

// 看板验证
- 看板名：1-100个字符，不能为空
